Secure storage, access control, and auditing  ensure sensitive information is protected and managed efficiently. Centralize key management to simplify compliance with security standards. Enhance the overall app security posture.</br> [Azure Key Vault](/azure/key-vault/general/overview) |
|**Optimal Maturity Status**</br></br> Agency continues to encrypt traffic as appropriate, enforces least privilege principles for secure key management enterprise-wide, and incorporates best practices for cryptographic agility as widely as possible. |**Key management in Azure**</br> The Azure key management services securely stores and manages cryptographic keys in the cloud, including Azure Key Vault, Azure Managed Hardware Security Model (HSM), and Azure Dedicated HSM. Azure enhances security, simplifies access control, supports applications, and protects sensitive data.</br> [Key management](/azure/security/fundamentals/key-management) </br></br>**Azure Key Vault**</br> Enforce least privilege principles through role-based access control (RBAC). Ensure only authorized entities access sensitive information.</br> - [Azure Key Vault best practices](/azure/key-vault/general/best-practices) </br> - [Grant permissions to apps to access Azure Key Vault](/azure/key-vault/general/rbac-guide)</br></br>**Microsoft Entra Privileged Identity Management**</br>Integrate Azure Key Vault with PIM for just-in-time (JIT) access. ## 3.4 Function: Network resilience

|**CISA ZTMM Stage Description**|**Microsoft guidance and recommendations**|
|---|--|
|**Initial Maturity Status**</br></br>Agency begins to configure network capabilities to manage availability demands for additional applications and expand resilience mechanisms for workloads not deemed mission critical. |**Azure Virtual Networks**</br>Adopt Azure to manage availability demands with its global network of datacenters and services.</br> - [Azure Virtual Network](/azure/virtual-network/virtual-networks-overview) </br> - [Azure reliability overview](/azure/reliability/overview)</br></br>**Availability zones**</br>To ensure apps are available, even if one zone has an outage, use availability zones for fault isolation in a region.</br> [Azure availability zones](/azure/reliability/availability-zones-overview) </br></br>**Azure ExpressRoute**</br>ExpressRoute is a hybrid connectivity service used for low latency, resilience, and high throughput private connectivity between an on-premises network and Azure workloads.</br> [ExpressRoute for resiliency](/azure/expressroute/design-architecture-for-resiliency)</br></br>|
|**Advanced Maturity Status**</br></br> Agency has configured network capabilities to dynamically manage the availability demands and resilience mechanisms for the majority of their applications. |**Azure Traffic Manager**</br> Dynamically distribute traffic across regions and datacenters. Ensure optimal performance and high availability; adapt to changing user-request patterns.</br> - [Traffic Manager](/azure/traffic-manager/traffic-manager-overview) </br> - [Reliability in Traffic Manager](/azure/reliability/reliability-traffic-manager)</br></br>**Azure Front Door**</br> Enhance global connectivity and security with dynamic HTTP/S load balancing and web application firewall services. |**Optimal Maturity Status**</br></br> Agency integrates holistic delivery and awareness in adapting to changes in availability demands for all workloads and provides proportionate resilience. |**Azure Load Balancer**</br> Configure Azure Load Balancer health probes to create awareness of application instance health status. Probes detect application failures, manage load, and adapt to availability demand changes.</br> - [Load Balancer health probes](/azure/load-balancer/load-balancer-custom-probe-overview) </br> - [Manage health probes](/azure/load-balancer/manage-probes-how-to)</br></br>**Azure Application Gateway**</br>Dynamically manage availability demand and resilience mechanisms by distributing traffic across multiple back-end pools and availability zones. Ensure high availability and fault tolerance, automatically reroute traffic during zone failures.</br> - [Azure Application Gateway v2](/azure/application-gateway/overview-v2) </br> - [Well-architected perspective](/azure/well-architected/service-guides/azure-application-gateway) </br></br>**Azure Firewall**</br> Enhance auto-scaling by automatically adjusting resources to meet traffic demands. Ensure security and performance under high loads. Use threat protection and URL filtering capabilities dynamically scale, based on throughput and CPU usage.</br> - [Premium features](/azure/firewall/premium-features) </br> - [Azure Firewall FAQ](/azure/firewall/firewall-faq) </br> - [Azure Firewall performance](/azure/firewall/firewall-performance)</br></br>**Azure ExpressRoute**</br>Configure Bidirectional Forwarding Detection (BFD) to improve ExpressRoute failover. Use rapid detection of link failures, and enable near-instantaneous switchover to back-up connections. Minimize downtime, maintain high availability, while making the network more resilient and reliable.</br> [Configure BFD](/azure/expressroute/expressroute-bfd)</br></br> |
